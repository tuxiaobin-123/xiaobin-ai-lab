今天花了大半天时间做技术选型，最后决定：**全部用 Next.js 单仓库**。

## 为什么不用 Framer？

Framer 视觉效果确实惊艳，但它是 no-code 工具，Claude Code 没有办法直接操作浏览器界面。如果主站用 Framer，意味着我每次改页面都得手动操作，30天高频迭代的话太麻烦了。

| 维度 | Framer | Next.js |
|------|--------|---------|
| 视觉效果 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| AI辅助开发 | ❌ | ✅ |
| 部署便捷性 | ✅ | ✅ Vercel |
| 自定义能力 | 有限 | 无限 |
| 学习成本 | 低 | 中 |

**结论**：个人品牌资产库的核心是功能，视觉可以用代码补回来。

## 单仓库 vs 多仓库？

三个工具共享：导航、Footer、颜色系统、API 工具函数。如果拆成三个仓库，每次改公共组件要改三次，维护成本翻三倍。**单仓库完胜**。

## 路由设计

```
/                    → 品牌主站
/prompt-gallery      → AI提示词画廊
/research            → 投研助手
/content-engine      → 内容引擎
/chronicles          → 实验日志（就是这里）
```

## 最重要的发现

现有的 `E:\person\` 项目已经有了不错的暗色主题设计，可以直接复用颜色系统，节省了2-3小时的设计决策时间。这就是"资产复用"的威力。
